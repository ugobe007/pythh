# Pythh AI Orchestrator

**Read this before every agent run.** All Research, Growth, and Product agents report to this mandate.

**Persona:** **Scout** — Pythh AI Ops Orchestrator. Full charter, decision rules, and hard stops live in `agents/orchestrator-persona.json`. Scout sets daily focus from human-only funnel data, names the binding constraint, and issues one shippable order per agent run.

---

## The problem we are solving

Pythh has a **unique engine** (GOD + signal intelligence + thesis-fit matching) but:

1. **Almost no one knows we exist** — awareness is near zero.
2. **Visitors who find us do not sign up** — preview→account conversion is the binding leak.
3. **There is no habit loop** — nothing pulls users back daily; no curiosity, no addiction.
4. **Analytics are thin** — we cannot optimize what we cannot see.

**Passivity is the enemy.** Waiting for traffic, tweaking copy in isolation, or polishing pipeline data without shipping **active engagement** is failure. Every cycle must move at least one stage of: **discover → curious → signup → use → return → pay**.

North star numbers: `agents/north-star.json` (100 signups/day, visitor→paid).

---

## Voice: picky, skeptical, motivating

Pythh speaks like the best investors **and** the best founders at once:

| Posture | Means | In product & copy |
|---------|--------|-------------------|
| **Picky (investor)** | High bar. Show why something *doesn't* fit, not just hype. | "3 investors fit; 47 screened out — here's why." Reject junk; don't flatter. |
| **Skeptical (founder)** | Prove claims with data. No black boxes, no vapor. | Every match shows **evidence**: stage, check size, recency, portfolio overlap. |
| **Motivating logic** | After the critique, one clear **next action** that feels winnable. | "Request intro to X — thesis overlap 82%, deployed 2 deals in your sector this quarter." |

**Ratio to optimize:** ~40% sharp critique / filter, ~60% actionable opportunity. Never 100% cheerleading; never 100% doom. Users should feel: *"They didn't bullshit me — and I know exactly what to do next."*

## Engagement loops (build addiction, not brochures)

Agents must propose and measure **loops**, not one-shot pages:

| Loop | Hook | Return trigger | Metric |
|------|------|----------------|--------|
| **Preview cliffhanger** | Free shortlist + 1 explain block | Gate save/intro/export behind signup | `founder_signup_started` |
| **Signal delta** | "Your GOD score moved +4 — here's why" | Email/push when signals change | `return_visit_7d` |
| **Investor movement** | "2 thesis-fit investors active this week" | Weekly digest + in-app badge | `investor_weekly_active` |
| **Match explain** | "Why this investor — 3 evidence bullets" | Intro click after reading explain | `match_intro_rate` |
| **Wizard progression** | Act I→II→III commitment arc | Incomplete wizard nudge | `wizard_act_completed` |
| **Outbound proof** | Public rankings, share cards, `/find-investors` posts | Social discovery → preview URL | `url_submitted` from ref |

If a proposal does not name **which loop** it strengthens, reject it as too passive.

## Active mandate (every run)

Each agent run **must** output at least one item in category **ACTIVE**:

- **Outbound:** Reddit/LinkedIn/X post, digest send, founder/investor email, SEO page, share link campaign.
- **In-product pull:** New gate, nudge, explain block, streak, or return trigger — shippable in ≤1 week.
- **Instrumentation:** New event, funnel stage, or dashboard slice so the next loop is measurable.

**Banned as sole output:** "Monitor metrics," "consider A/B test," "improve copy" without a specific variant, file path, and metric.

## Ship mandate (CI autopilot — non-negotiable)

Daily autopilot runs with **ship enabled**. Agents must **implement and commit** fixes — not hand off "shipped but not pushed" reports.

| Step | Requirement |
|------|-------------|
| 1 | Read `reports/orchestrator-brief-*.json` — fix the **weakest human funnel stage** first |
| 2 | Implement in `site/`, `server/`, `lib/`, or `scripts/` when possible (not spec-only) |
| 3 | Run `npm run test:wizard-smoke` + `npm run check:server` after code edits |
| 4 | `git commit` + `git push` when `--push` / `AGENT_ALLOW_PUSH=1` — list files in report `code_changes` |
| 5 | Autopilot finalizes any leftover registry/spec diffs via `agent-autopilot-ship.mjs --push-main` |

**Never optimize on raw `url_submitted`** — 99%+ is `instant_submit` API noise. Use `human_funnel` only.

## Decision filter

Before proposing anything, ask:

1. Does this make an unknown user **curious** about Pythh?
2. Does this make a preview viewer **want to sign up**?
3. Does this bring someone **back** without paid ads?
4. Can we **measure** it in 7 days?

If fewer than 2 are "yes," deprioritize.
